01 — grunder
Grunder
HTML beskriver innehåll med taggar. En tagg omsluter innehåll med en öppnings- och en stängningstagg.
tagg
Öppningstagg, innehåll, stängningstagg. Stängningstaggen har ett snedstreck.
<p>Det här är ett stycke.</p>

<h1>Det här är en rubrik.</h1>
självstängande tagg
Vissa taggar har inget innehåll och stänger sig själva — de har ingen stängningstagg.
<img src="https://picsum.photos/400/300">
<br>
<hr>
attribut
Extra information i öppningstaggen. Alltid i formatet namn="värde".
<a href="https://example.com">Klicka här</a>

<img src="https://picsum.photos/400/300" alt="En slumpmässig bild">

<div class="rubrik" id="toppen">Innehåll</div>
02 — dokumentstruktur
Dokumentstruktur
Varje HTML-fil börjar med en fast grundstruktur. Webbläsaren förväntar sig den här uppbyggnaden.
grundmall
En minimal men komplett HTML-fil.
<!DOCTYPE html>
<html lang="sv">
  <head>
    <meta charset="UTF-8">
    <title>Sidans titel</title>
  </head>
  <body>
    <p>Här börjar innehållet.</p>
  </body>
</html>
<!DOCTYPE html>
Talar om för webbläsaren att det är HTML5. Alltid första raden.
<html>
Rotelementet — allt annat ligger inuti den. lang="sv" anger språket.
<head>
Metadata som inte syns på sidan: titel, teckenkodning, länk till CSS-filer.
<meta charset="UTF-8">
Anger teckenkodning. UTF-8 stödjer svenska tecken (å, ä, ö). Alltid med.
<title>
Sidans titel — syns i webbläsarfliken, inte på sidan själv.
<body>
Allt synligt innehåll på sidan placeras här.

04 — divvar & klasser
Divvar & klasser
div är ett neutralt behållarelement utan egen betydelse. Klasser och id:n gör att CSS kan hitta rätt element och ge det ett utseende.
div
En osynlig låda för att gruppera element. Används nästan alltid med en class.
<div>
  <p>Den här texten sitter i en div.</p>
  <p>Den här också.</p>
</div>
class
Namnger ett element så att CSS kan ge det ett utseende. Flera element kan ha samma klass. I CSS används ett punktprefix: .klassnamn
<div class="kort">Första kortet</div>
<div class="kort">Andra kortet</div>
Flera klasser på samma element är möjligt — separera dem med mellanslag. Elementet får då stilarna från alla klasser kombinerat. 
<div class="kort markerad stor">Tre klasser</div>
Hur man skriver CSS-regler för klasser förklaras i CSS → Grunder ↗
id
Unikt namn på ett enskilt element. Ska bara finnas ett per sida. I CSS används ett #-prefix: #idnamn
<div id="sidhuvud">Logga och nav</div>
05 — nesting
Nesting
Taggar kan ligga inuti varandra — det kallas nesting. Det skapar en trädstruktur av element med tydliga släktskapsförhållanden.
förälder & barn
När ett element ligger direkt inuti ett annat är det yttre föräldern och det inre barnet. Ett element kan ha många barn, och barnen är då syskon till varandra.
<div class="behållare">   <!-- förälder -->
  <div class="kort">Barn 1</div>   <!-- barn, syskon -->
  <div class="kort">Barn 2</div>   <!-- barn, syskon -->
  <div class="kort">Barn 3</div>   <!-- barn, syskon -->
</div>
förälder — div.behållare
barn 1
barn 2
barn 3
flera nivåer
Nesting kan gå hur djupt som helst. Varje element stängs alltid innan sin förälder stängs.
<div class="sida">              <!-- farförälder -->
  <div class="artikel">         <!-- förälder -->
    <h2>Rubrik</h2>             <!-- barn -->
    <p>Text med <strong>fet</strong> del.</p>  <!-- barn -->
  </div>
</div>
varför det spelar roll i CSS
CSS-egenskaper som color och font-size ärvs av barnen. Layout-egenskaper som display: flex på föräldern styr hur barnen placeras.
Läs mer om hur förälder/barn används i CSS: CSS → Förälder & barn ↗
01 — grunder
Grunder
En CSS-regel består av en selector, ett block med klamrar, och ett eller flera deklarationer. Varje deklaration avslutas med semikolon.
h1  {  color  :  #c0392b ;  }
selector — vilket element property — vad som ändras value — det nya värdet semikolon — avslutar varje rad
selector — element
Matchar alla element av den typen på sidan.
p {
  color: #333333;    /* all <p>-text får den här färgen */
}

h1 {
  font-size: 32px;   /* alla <h1> får den här storleken */
}
selector — klass
Matchar element med class="klassnamn". Börjar alltid med en punkt.
.rubrik {
  font-weight: 700;
}

.kort {
  background-color: #ffffff;
  border: 1px solid #dddddd;
}
selector — id
Matchar elementet med id="namn". Börjar alltid med #. Bara ett per sida.
#sidhuvud {
  position: sticky;
  top: 0;
}
flera regler, semikolon
Man kan ha hur många deklarationer som helst i ett block. Varje rad avslutas med semikolon — utan det tolkas nästa rad som en fortsättning.
.kort {
  background-color: #ffffff;
  border: 1px solid #dddddd;
  border-radius: 8px;
  padding: 16px;
  color: #1c1917;
}
kommentar
Text som ignoreras av webbläsaren. Bra för anteckningar i koden.
/* det här är en kommentar */

.kort {
  padding: 16px; /* inre luft */
}
02 — enheter
Enheter
CSS har flera olika måttenheter. Välj rätt enhet beroende på om du vill ha ett fast eller relativt mått.
px — pixel
Fast absolut storlek. Påverkas inte av förälderns storlek. Bra för ramar och exakta mått.
font-size: 16px;
border: 2px solid #333333;
width: 300px;
em
Relativt till förälderns teckenstorlek. Staplas ihop vid nesting — ett element på 1.2em inuti ett element på 1.2em blir 1.44× grundstorleken.
font-size: 1em;    /* samma som föräldern */
font-size: 1.25em; /* 25% större än föräldern */
padding: 0.5em;    /* halva den aktuella textstorleken */
letter-spacing: 0.08em;
rem — root em
Relativt till rotelementets teckenstorlek — vanligtvis 16px. Staplas inte ihop. Bra för konsekvent typografi på hela sidan.
font-size: 1rem;     /* 16px (standard) */
font-size: 1.5rem;   /* 24px */
font-size: 0.875rem; /* 14px */
% — procent
Relativt till förälderns motsvarande egenskap. Används ofta för bredd och höjd.
width: 100%;    /* lika bred som föräldern */
width: 50%;     /* halva förälderns bredd */
height: 100%;   /* kräver att föräldern har en definierad höjd */
pt — punkt
Tryckerienhet. Används sällan på webben. Föredra px eller rem för skärm.
font-size: 12pt; /* motsvarar ungefär 16px — används sällan på webben */
vh & vw — viewport
Relativt till fönstrets synliga yta. 1vh = 1% av fönstrets höjd, 1vw = 1% av fönstrets bredd. Används för element som ska fylla hela skärmen oavsett skärmstorlek.
height: 100vh;      /* exakt lika hög som fönstret */
min-height: 100vh;  /* minst lika hög som fönstret, kan växa */
width: 100vw;       /* exakt lika bred som fönstret */
height: 50vh;       /* halva fönsterhöjden */
Rekommendation: använd rem för teckenstorlekar, px för borders och spacing, och % eller rem för layoutbredder. Undvik pt och em för font-size tills du förstår hur staplingseffekten fungerar.
03 — boxmodellen
Boxmodellen
Varje element är en rektangulär låda. Lagren staplas utåt: innehåll → padding → border → margin.
Verktyg → Boxmodell-visualiserare ↗
margin
border
padding
innehåll
margin— utanför ramen
border— synlig ram
padding— inuti ramen
innehåll— text, bilder, barn
margin
Utrymme utanför elementet. Kortformen går topp → höger → botten → vänster.
margin: 8px;                /* alla sidor */
margin: 8px 16px;           /* topp+botten  vänster+höger */
margin: 8px 16px 8px 16px; /* topp höger botten vänster */
margin-top: 24px;           /* en sida i taget */
margin: auto;               /* centrera horisontellt */
padding
Utrymme mellan innehållet och ramen. Samma kortform som margin.
padding: 12px 20px;
padding-bottom: 32px;
border
Synlig ram runt elementet. Tar bredd, stil och färg. border-radius rundar hörnen — tillräckligt stort värde ger en cirkel.
border: 2px solid #333333;
border-top: 1px dashed #cccccc; /* en sida i taget */

border-radius: 4px;    /* lätt rundade hörn */
border-radius: 12px;   /* tydligt rundade hörn */
border-radius: 999px;  /* piller-form (fullt rundad) */
border-radius: 50%;    /* cirkel (kräver lika bredd och höjd) */
solidheldragen linje
dashedstreckad linje
dottedprickad linje
doubledubbel linje
noneingen ram
Rekommendation: solid är den överlägset vanligaste stilen. dashed och dotted används ibland för avgränsare. Undvik double — den ser sällan bra ut.

04 — förälder & barn
Förälder & barn
Begreppen förälder, barn och syskon beskriver hur element är nästlade i varandra. I CSS är det avgörande — layoutsystem som Flexbox styrs från föräldern och påverkar alla dess barn.
vad begreppen innebär
Föräldern är elementet som omsluter. Barnen är elementen direkt inuti. Barn på samma nivå kallas syskon.
Nesting och förälder/barn-relationer förklaras med exempel i HTML-guiden: HTML → Nesting ↗
arv & layout
Föräldern styr hur barnen placeras. Egenskaper som color och font-size ärvs automatiskt av barnen.
/* föräldern styr layout och delar ut gemensamma stilar */
.behållare {
  display: flex;
  gap: 16px;
  color: #333333; /* ärvs av barnen */
}

/* barnet stylar bara sig självt */
.kort {
  padding: 16px;
  background-color: #ffffff;
}
05 — positionering
Positionering
Styr var ett element hamnar på sidan — i det normala flödet, utanför det, eller fastnaglat vid skärmkanten.
position
Grundegenskapen. Aktiverar top, right, bottom, left.
position: static;   /* standard — följer normalt flöde */
position: relative; /* förskjuts från sin normala plats */
position: absolute; /* positioneras mot närmaste icke-statiska förälder */
position: fixed;    /* fastnaglat i fönstret, scrollar inte med */
position: sticky;   /* följer med vid scroll tills det når en angiven gräns */
staticstandard — följer sidans normala flöde
relativeförskjuts från sin normala position
absolutelyfts ur det normala flödet, placeras mot närmaste icke-statiska förälder
fixedfastnaglat i fönstret, påverkas inte av scroll
stickybeter sig som relative tills det når den angivna övre eller nedre gränsen
Rekommendation: sticky för sidhuvuden, relative när du behöver förskjuta något lite. Använd absolute sparsamt — det lyfter elementet ur flödet och kan skapa oväntade överlapp.
top, right, bottom, left
Används tillsammans med position (ej static) för att förflytta elementet.
/* sticky sidhuvud */
position: sticky;
top: 0;

/* fastnaglat i nedre högra hörnet */
position: fixed;
bottom: 24px;
right: 24px;

/* absolut placerad inuti en relativ förälder */
position: absolute;
top: 0;
left: 0;
z-index
Styr stapelordning när element överlappar. Högre värde hamnar ovanpå. Fungerar bara på element som inte har position: static.
z-index: 0;    /* standard */
z-index: 10;   /* ovanpå element med lägre värde */
z-index: 100;  /* typiskt för sidhuvud */
z-index: 999;  /* typiskt för modaler och tooltips */
width & height
Anger elementets storlek. min- och max-varianter sätter gränser istället för fasta mått.
width: 300px;      /* fast bredd */
width: 100%;       /* lika bred som föräldern */
max-width: 600px;  /* aldrig bredare än 600px */
min-width: 200px;  /* aldrig smalare än 200px */

height: 200px;
min-height: 100vh; /* minst hela fönsterhöjden */
Vad är vh? Se Enheter → vh & vw ↗
display
Styr hur ett element beter sig i flödet.
display: block;        /* tar upp hela raden, staplas vertikalt */
display: inline;       /* flödar med text, tar bara nödvändig plats */
display: inline-block; /* som inline men accepterar bredd och höjd */
display: none;         /* döljer elementet helt — tar ingen plats */
display: flex;         /* aktiverar flexbox-layout */
blockdiv, p, h1 — tar hela raden
inlinespan, a, strong — rinner med i texten
inline-blocksom inline men med bredd/höjd
nonedöljer elementet, tar ingen plats
flexaktiverar flexbox
Rekommendation: när du vill placera element bredvid varandra, använd display: flex på föräldern. Det är nästan alltid bättre än inline-block.
float
Lyfter elementet ur det normala flödet och låter omgivande text ringla runtom det. Mest historisk — Flexbox är nästan alltid bättre.
float: left;   /* elementet flyter till vänster, text flödar till höger */
float: right;  /* elementet flyter till höger */
float: none;   /* standard */
Rekommendation: undvik float för layout. Det var standardlösningen före Flexbox men orsakar lätt oväntade buggar. Använd det bara om du vill låta text flöda runt en bild.
06 — flexbox
Flexbox
Endimensionell layout. Sätt display: flex på föräldern — barnen placeras automatiskt i rad eller kolumn.
Verktyg → Flex-lekplats ↗
förälder — display: flex
barn 1
barn 2
barn 3
Föräldern styr med:
display, flex-direction, justify-content,
align-items, flex-wrap, gap
Barnen styr med:
flex, flex-grow, flex-shrink, flex-basis,
align-self
På föräldern
display: flex
Gör elementet till en flex-behållare. Barnen blir flex-element.
display: flex;
flex-direction
Sätter huvudaxelns riktning. Standard är rad (horisontellt).
flex-direction: row;            /* → standard */
flex-direction: row-reverse;    /* ← */
flex-direction: column;         /* ↓ */
flex-direction: column-reverse; /* ↑ */
justify-content
Justerar barnen längs huvudaxeln.
justify-content: flex-start;    /* standard — barnen samlas i början */
justify-content: flex-end;      /* barnen samlas i slutet */
justify-content: center;        /* barnen centreras */
justify-content: space-between; /* mellanrum mellan barnen, inga yttermarginaler */
justify-content: space-around;  /* lika luft runt varje barn */
justify-content: space-evenly;  /* exakt lika mellanrum överallt */
align-items
Justerar barnen längs korsaxeln (vinkelrätt mot huvudaxeln).
align-items: stretch;    /* standard — barnen sträcks ut */
align-items: flex-start; /* barnen justeras mot startkanten */
align-items: flex-end;   /* barnen justeras mot slutkanten */
align-items: center;     /* barnen centreras */
align-items: baseline;   /* barnen justeras efter textbaslinjen */
flex-wrap
Låter barnen radbryta sig istället för att rinna utanför kanten.
flex-wrap: nowrap;       /* standard — ingen radbrytning */
flex-wrap: wrap;         /* barnen radbryts vid behov */
flex-wrap: wrap-reverse; /* radbryts omvänt */
gap
Avstånd mellan barnen. Snyggare än marginaler på enskilda barn.
gap: 16px;       /* samma avstånd i båda riktningar */
gap: 12px 24px;  /* rad-gap  kolumn-gap */
På barnen
flex
Kortform för grow, shrink och basis — styr hur ett barn dimensionerar sig relativt sina syskon.
flex: 1;           /* väx för att fylla tillgängligt utrymme */
flex: 0 0 200px;   /* grow  shrink  basis — fast 200px */
flex-grow: 1;      /* ta upp ledigt utrymme */
flex-shrink: 0;    /* krymper aldrig under basis */
flex-basis: 200px; /* startstorlek innan grow/shrink */
align-self
Åsidosätter förälderns align-items för just det här barnet.
align-self: flex-end;
align-self: center;
align-self: stretch;
07 — typografi
Typografi
Importera ett typsnitt från Google Fonts eller använd web-säkra typsnitt som alltid finns tillgängliga.
Verktyg → Typsnittsprovare ↗
font-family
Anger vilket typsnitt som används. Skriv alltid en lista med flera alternativ — webbläsaren provar dem i ordning och använder det första den hittar. Sist i listan ska alltid ett generiskt reservtypsnitt stå.
font-family: 'Inter', Arial, sans-serif;
/*            ^importerat  ^web-säkert  ^generiskt */
sans-serifutan seriffer — rent och modernt (Arial, Helvetica)
serifmed seriffer — klassiskt och läsbart i tryckt stil (Georgia, Times)
monospacefast teckenbredd — används för kod (Courier New)
web-säkra typsnitt
Dessa typsnitt finns förinstallerade på i princip alla datorer och telefoner — ingen import behövs. Bra att ha som reserv efter ett importerat typsnitt.
TypsnittTypBra reserv förExempeltext

/* det du vill ha -> web-säkert som liknar -> generisk sist */
font-family: 'Lora', Georgia, serif;
font-family: 'Inter', Arial, sans-serif;
font-family: 'Fira Code', 'Courier New', monospace;
Rekommendation: Georgia är det snyggaste av de inbyggda — bättre än Times New Roman för löptext. Vill du ha ett mer personligt utseende, importera från Google Fonts (se nedan).
importera från Google Fonts
Ger tillgång till tusentals typsnitt. Lägg @import överst i din CSS-fil — före alla andra regler. Välj typsnitt och kopiera koden på fonts.google.com.
/* överst i din CSS-fil, före allt annat */
@import url('https://fonts.googleapis.com/css2?family=Lora:wght@400;700&display=swap');

/* sedan använder du typsnittsnamnet */
font-family: 'Lora', Georgia, serif;
font-size
rem är relativt till rotelementet (oftast 16px). em är relativt till föräldern.
font-size: 16px;   /* absoluta pixlar */
font-size: 1rem;   /* relativt till root — 16px som standard */
font-size: 1.25em; /* 1,25× förälderns teckenstorlek */
font-weight
Värden från 100 till 900. Bara vikter som typsnittet faktiskt innehåller fungerar.
font-weight: 400; /* normal */
font-weight: 600; /* halvfet */
font-weight: 700; /* fet */
font-weight: 800; /* extra fet */
line-height
Enhetslösa värden skalas med teckenstorleken — föredra dem framför px.
line-height: 1;    /* tight — bra för rubriker */
line-height: 1.5;  /* bekvämt för brödtext */
line-height: 1.75; /* luftigt */
Rekommendation: använd 1.5–1.6 för brödtext — det är det intervall som är lättast att läsa. Rubriker mår bättre av 1–1.2.
letter-spacing
em skalas med teckenstorleken och är den naturliga enheten här.
letter-spacing: -0.02em; /* tight — bra för stora rubriker */
letter-spacing: 0;       /* normal */
letter-spacing: 0.12em;  /* brett — bra för etiketter och versaler */
text-align
Horisontell justering av text inuti sin behållare.
text-align: left;    /* standard */
text-align: center;
text-align: right;
text-align: justify; /* sträcker ut rader till full bredd */
text-transform
Ändrar versalisering utan att röra HTML-koden.
text-transform: uppercase;  /* VERSALER */
text-transform: lowercase;  /* gemener */
text-transform: capitalize; /* Första Bokstaven I Varje Ord */
text-transform: none;       /* standard */

background
Solid färg eller bild, med kontroll över storlek, position och upprepning.
background-color: #f6f3ef;
background-image: url('foto.jpg');
background-size: cover;       /* fyll ytan, beskär vid behov */
background-size: contain;     /* passa in, kan ge tomma kanter */
background-position: center;
background-repeat: no-repeat;
coverfyll hela ytan, beskär om nödvändigt
containvisa hela bilden, kan ge tomma kanter
autobildens originalstorlek
no-repeatbilden visas en gång
repeatbilden upprepas som mönster (standard)
linear-gradient
Mjuk färgövergång längs en riktning. Lägg till fler färger för fler stopp.
background: linear-gradient(90deg, #ff7700, #ffcc00);
background: linear-gradient(to bottom, #ffffff, #eeeeee);

/* flera färgstopp */
background: linear-gradient(135deg, #7c3aed, #2563eb, #22cc66);
radial-gradient
Strålar utåt från en mittpunkt.
background: radial-gradient(circle, #2563eb, #000000);
background: radial-gradient(ellipse at top, #7c3aed, #000000);
